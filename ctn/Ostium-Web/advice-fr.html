Les conseils suivants ne concernent pas spécifiquement l'OSINT mais plutôt votre vie privée.

Considérer le navigateur OOB comme un outil. Ne l'utilisez pas pour un usage privé !

Pourquoi ? Cela vous permettra de ne jamais croiser vos investigations et vos données privées. Assurez-vous de ne rien faire qui puisse vous identifier, garantissant ainsi l'intégrité de votre navigateur, où aucune trace de votre activité privée ne subsistera. Même si OOB permet à chaque fermeture de complètement écraser les données de navigations avec une méthode où ni Edge Chrome ni Google Chrome n'a de pareil, cette action se réalise en fin de session. Donc si par exemple vous consultez un mail privé pendant une investigation vous risquez de laisser une trace sur le serveur visité pouvant vous identifier. Si en retour vous n'utilisez que des mails spécifiques à l'usage de vos investigations, mails qui eux même ne sont jamais utilisés pour des actions privées, alors vous amélioré les possibilités de ne pas laisser de trace vous identifiant avec votre navigateur. C'est le même principe pour les réseaux sociaux si pendant une investigation vous utilisez un de vos réseaux qui utilise des infos privées cela sera une possibilité de laisser une trace permettant de vous identifier.

Bien sûr vous pouvez créer différents répertoires distincts les uns des autres et y placer le répertoire d'installation d'Ostium qui fonctionne en mode portable. Les données d'utilisations ne se croisent pas et sont situées chacune dans un environnement indépendant il est donc possible d'utiliser une version d'Ostium spécifique à chacun de vos besoins et donc à la navigation privée qui ne croisera pas avec la recherche, mais il est déconseillé d'utiliser OOB pour croiser toutes vos activités. Vous devriez d'ailleurs faire de même avec tous navigateurs confondus. Avec OOB En fin de session l'écrasement des données et comparable à une réinstallation complète rien n'est conservé à l'exception des données que vous collectés dans vos investigations. Côté sécurité Ostium profite des mêmes avantages que Edge Chrome, en effet Ostium repose sur le runtime Microsoft WebView2 qui reçoit des mises à jour quotidienne de Microsoft.

Il est crucial de segmenter vos usages sur le Web. Utilisez une adresse mail différente pour chaque service que vous utilisez. Par exemple, utilisez une adresse unique spécifiquement pour les achats en ligne, et ne l'utilisez jamais ailleurs qu'à des fins de paiement. Ne reliez jamais aucun compte entre eux. Par exemple, ne croisez pas vos données Steam avec celle d'Epic soyez une entité différente pour chacun d'entre eux et les autres. Idem pour Discord créez un compte pour chaque serveur ou divisez plusieurs serveurs sur différents comptes selon vos besoins. Si vous êtes actifs sur des forums utilisez différents comptes et adresses e-mail pour chacun d'entre eux. Si vous disposez d'un serveur mail il est utile de créer des adresses temporaires pour chaque investigations, services etc. Adresse que vous supprimez à la fin de votre enquête. Vous pouvez utiliser de nombreux services en ligne pour la création de vos mails mais rien ne garantit la temporalité des adresse créés et de leur suppression réelle voir leur non-réutilisations. Il est important de toujours garder la main un maximum et faire appel aux outils tiers qu'en cas d'incapacité, je recommande toujours en première phase les outils propriétaires, codés en interne.

Prenez l'habitude de varier entre différents navigateurs selon les usages. Par exemple, j'utilise les outils Microsoft pour les services Microsoft, Chrome avec Gmail, Firefox pour les sites underground, OOB pour la recherche approfondie et Tor pour ses usages spécifiques. Sur YouTube, je possède un compte pour chaque thématique de vidéos que je regarde, exemple un compte pour les vidéos qui traitent de code et de programmation, un compte loisir pour écouter de la musique et regarder des vidéos divertissantes etc. J'utilise aussi différents comptes de chaînes selon la thématique et mes besoins, évitant ainsi la corrélation de données par les trackers et autres. Sur le Web il faut se diversifier tout comme l'activé y est diversifiée. C'est le moyen le plus simple pour ne pas être reconnu comme une entité définie mais plutôt un ensemble de probabilités. Il faut apprendre à être une personnalité multiple dans votre utilisation du réseau.

Pas besoin de préciser qu'il est important pour que tout cela soit efficace d'utiliser un VPN et autre méthode d'anonymisation. Je précise que je n'ai pas confiance au VPN. Pour ce qui est des pseudos faites en sorte d'en posséder de nombreux et qui n'ont pas de similitude. Pensez aussi à varier votre style d'écriture pour éviter les techniques d'analyses syntaxique qui peuvent vous identifier facilement. Vous pouvez vous créer une base de données de commentaire réel et générique vous permettant d'apporter des réponses dont la syntaxe et totalement différente de la vôtre, l'IA est un outil très utile pour cela, faites aussi attention au mot rare ou fautes d'écritures particulières le mieux étant d'en générer de fausses. Prenez l'habitude de ne jamais utiliser votre vraie identité ni âge, ni rien sauf si cela s'avère obligatoire. Pareil pour votre langue, utilisé l'anglais en toutes situations, idem pour les apps les services Web etc. configurez tout en langue anglaise ou autre voir différentes langues selon les services. Sur PC utilisez un clavier Qwerty physiquement.

Si vous partagez un téléphone, utilisez une ligne dédiée. Ne partagez jamais votre téléphone personnel laissez-le en dehors du réseau. Si vous avez l'opportunité, utilisez 3 téléphones : Privée et pro / Social et amis / Achat et administratif.

Considérer tout le monde comme suspect et ne communiquez que si cela s'avère obligatoire, ne transmettez jamais d'infos personnel dans une discussion. N'oubliez pas dans une communication avec une source de confiance il peut s'y trouver l'homme du milieu. Connectez votre micro et votre caméra seulement si vous les utilisez. Ne gardez pas votre téléphone dans la pièce où vous vous situé lorsque vous n'en avais pas un réel besoin. Garder le à une distance audible pour les notifications mais suffisamment inaudible pour vos discussions privées. Si vous souhaitez utilisez un logiciel de sécurisation pour vos mots de passe créez le vous-même ou n'en utilisez pas et préférez la mémorisation.

Ne pensez pas qu'un outil Open Source sur GitHub est une garantie de sécurité. A l'heure de l'écriture de ce texte 0.3% de la population mondiale sait coder (sources : cabinet IDC), on est comme au temps de l'illettrisme ou peu était les gens qui savaient lire, donc peu sont ceux qui ont les aptitudes nécessaires pour faire un audit de code afin de vérifier si une backdoor ou autre existe. Les exemples de sociétés qui se sont faites piéger par du code Open Source sont nombreux. Basez-vous donc sur des critères plus censés quand vous utilisez des outils tiers. Idem ne pensez pas que par ce qu'un code est fermé qu'il est douteux je ferme souvent mon code cela pour des raisons de propriété intellectuel et aussi pour éviter qu'un Script kiddie s'approprie mon code pour faire du sale. La seule fiabilité c'est le code qu'on rédige soit même, rien d'autre à attendre.

Prenez l'habitude pour vos données personnelles d'utiliser des supports de stockages externes et facilement destructible que vous ne connectez qu'en cas de besoins.

Pour finaliser, important ne pensez jamais sécurité mais probabilité, la sécurité n'existe pas sur le Web. Un ordinateur sécurisé est celui qui reste dans sa boîte, jamais allumé (et encore le fabriquant reste un élément à considérer). En dehors de ça vous ne faites qu'essayer de diminuer les probabilités d'une atteinte à la sécurité de vos données, système etc.

Tout ce qui est dit ici n'est pas exhaustif. Il y'a encore énormément à faire pour se sécuriser sur le réseau. La meilleur étant de ne pas s'y trouver.
